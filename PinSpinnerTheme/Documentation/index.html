Introduction

After unzip the package, you need the HTML folder content. There are HTML, CSS, Javascript and SCSS files used in the template. Create a new folder with these files to start editing them. The template is based on the Bootstrap Framework v4. The complete documentation on this framework can be found at http://getbootstrap.com/.

Customizing

You can customize the template in many ways.

Via CSS

In the folder assets/css there is a file named custom.css. Use it to override the template styles to fit your needs like the example below: 

/* Default style in template.css*/
body {
  color: #666666;
}

/* Your style in custom.css */
body {
  color: #222222;
}

Template main color

The variable $template-main-color is the default template color. This will be the default color of buttons, links and other elements on the page. Use any color in the color palette or your preferred color.

Margins and Paddings

AppX uses the CSS classes of shorthand responsive margin and padding utility classes to modify an element's appearance. For example: 

<p class="lead mx-auto mt-5">
    Our app can bring many benefits to you. See below some of them.
</p>

  • mx-auto is used to center the element horizontally
  • mt-5 gives a level 5 margin to the paragraph
More details at Bootstrap Spacing

Navigation

Fixed navbar

AppX has a fixed navigation with brand text, a centralized menu, and a search button on the right. You can edit the navigation menu via CSS or SCSS. You can define a new position: 

$("#navigation").fixed({
    topSpace: 50
});

Scrollspy behavior

The navigation automatically update menu links based on scroll position. You can change the scroll speed or change the offset: 

$("#navigation").scrollspy({
    scrollSpeed: 1000,
    offset: 100
});

Mobile breakpoint

The variable mobileBreakpoint is used to define when the navigation goes to the portrait mode. 

$("#navigation").navigation({
    mobileBreakpoint: 767
});

mobileBreakpoint

If you change the variable mobileBreakpoint you may need to change the grid breakpoint.

Google Maps

Configuring the map to your location

We use the "JQuery Location Picker" plugin to make easy to use Google Maps API. Along with single point selection, it allows area selection by choosing a point and specifying a radius. In main.js you can see the plugin initialization: 

$('#map').locationpicker({
    location: {
        latitude: 48.8588208,
        longitude: 2.3495592
    },
    radius: 10
});

Replace by your coordinates and a desired radius value. For information about the plugin go to JQuery Location Picker. For information about Google Maps API go to Google Maps Javascript API.


Google Maps API

It's ideal to use the Google Maps API with an API key to avoid limitations in using maps. Check the link above for how to get your API key.

Preloader

To make the image loading more friendly, we use the "imagesLoaded" plugin. This plugin shows a CSS animation until all images on the page are loaded. The first element inside page's body is the "loader": 

<!-- Loader -->
<div class="loader">
    <div class="cp-spinner cp-skeleton"></div>
</div>

In main.js: 

$("body").imagesLoaded().always(function(instance){
    $(".loader").delay(300).fadeOut(500);
});

For more information about "imagesLoaded" plugin go to imagesLoaded.
